Recycler 自定义 ItemDecoration

发表于 2016-05-20   |   分类于 Android   |  

简介

RecyclerView 在绘制的时候会调用 ItemDecoration ，主要被用来绘制分割线。

GoogleSamples有相关的 示例代码。

方法介绍

ItemDecoration 中的三个方法

getItemOffsets(Rect outRect, View view, RecyclerView parent, State state)

实现该方法需要在传入的 outRect 中设置要绘制的分割线的宽度，可以理解为四个值分别为四面的分割线宽度，最后会以 padding 的形式加到 item 上面。

samples 解析

通过对构造方法中传入的 orientation 判断后设置 padding ，如果是竖直布局，则在 item 下面设置 padding ，如果是水平布局，则在 item 右侧设置 padding。

自己实现

这里可以通过 RecyclerView.LayoutParams 获取到当前 item 的 position ，也可以通过 RecyclerView 获取到当前的 LayoutManager ，对其判断后设置左上右下的 padding ，比直接在布局文件中设置更为灵活。

onDraw(Canvas c, RecyclerView parent, State state)

该方法会在绘制 item 之前调用，绘制范围是 RecyclerView 范围内的任意位置，不局限在 item 中。

samples 解析

通过遍历当前 RecyclerView 的 childView ，通过 drawable.setBounds() 设置绘制的范围，调用 draw() 进行绘制，在每个 childView 旁绘制需要的分割线。
以竖直布局为例，需要绘制的区域的 left 和 right 是固定的，而 top 和 bottom 值是不同的，每个 item 都有对应的值。

  • left: parent 的 paddingLeft 值，这里可能有些不合适，我觉得写成 parent.getLeft() + parent.getPaddingLeft() 会好一点
  • right: RecyclerView 右侧边界
  • top: 由于该例子中每个 item 对应的分割线在 item 下面，所以绘制的top值为 child.getBottom() + params.bottomMargin + Math.round(ViewCompat.getTranslationY(child))
  • bottom: top 的值加上 divider 的高

这里要注意的是由于 onDraw 是在最下面绘制的，避免过度绘制，被 item 挡住的部分就不要绘制了。

onDrawOver(Canvas c, RecyclerView parent, State state)

该方法会在 item 绘制完之后调用，绘制在最上层。使用方法和 onDraw() 相同。

总结

总体来看还是比较简单的，通过 getItemOffsets() 来设置额外的 padding ，再通过 onDraw() 或者 onDrawOver() 进行绘制。
这个类虽然说是用于绘制分割线，但是可以看到，我们可以为每个 item 根据条件不同的 padding ，也可以通过 onDrawOver() 随便在 RecyclerView 上的任何地方绘制任何东西，而不仅仅局限于分割线，至于可以绘制其他的什么，就需要大开脑洞啦。
